Delphi Indy10坑 之 TIdHTTPServer无法接收Post过来的值

Posted on 2016-10-28  |  2016-11-14   |   In Delphi

由于软件需求,需要在本地开放一个HttpServer. 然后通过web传值过来 然后进行xxxx操作

环境

  • jquery 2.0 + ajax + post
  • delphi 7 + indy 10 + TIdHTTPServer

Bug

前端Post后 delphi-HttpServer->Request.Params 中是空的 (没有获取到参数)

原因

跟踪源码发现

if ParseParams then begin
    //if LowerCase(LRequestInfo.ContentType) = 'application/x-www-form-urlencoded' then begin    {Do not Localize}
    if (pos('application/x-www-form-urlencoded',(LowerCase(LRequestInfo.ContentType)))&gt;0) then begin    {Do not Localize}
        LRequestInfo.DecodeAndSetParams(LRequestInfo.UnparsedParams);
    end
    else begin
        // Parse only query params when content type is not 'application/x-www-form-urlencoded'    {Do not Localize}
        LRequestInfo.DecodeAndSetParams(LRequestInfo.QueryParams);
    end;
end;

注释掉的//if LowerCase(LRequestInfo.ContentType) = 'application/x-www-form-urlencoded' then是原本的源码

我们可以看出 它是比较ContentType的内容.

如果ContentType是application/x-www-form-urlencoded,那么解析UnparsedParams.


但是我们传过来的ContentType的内容是application/x-www-form-urlencoded;charset: UTF-8 这就导致等式不成立 从而解析了QueryParams.

解决方法

将 LowerCase(LRequestInfo.ContentType) = ‘application/x-www-form-urlencoded’
改成 pos(‘application/x-www-form-urlencoded’,(LowerCase(LRequestInfo.ContentType)))>0
