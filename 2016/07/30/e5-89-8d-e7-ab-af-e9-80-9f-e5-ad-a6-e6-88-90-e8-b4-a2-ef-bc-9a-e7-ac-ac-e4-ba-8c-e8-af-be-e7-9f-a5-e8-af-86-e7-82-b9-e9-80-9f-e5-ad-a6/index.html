前端速学成财：第二课-知识点速学

Veröffentlicht am 2016-07-30  |  2016-11-14   |   in 前端专区 . 前端速学成财

1、学会外部文件引用

2、NPM的初步使用

3、自己创建一个模块

1.基本语法(不用理解,死记硬背即可):

外部引用时 建议写法:
exports.函数名 = function()
{
    //xxoo
}
或
module.exports.函数名
`</pre>

新建文件 aa.js(后缀随意,通常用js)

<pre>`    var getlib=require('./slib');
    getlib.showName();
    console.log(getlib.age);
`</pre>

新建文件 slib.js(后缀随意,通常用js)

<pre>`    exports.age=20;
    exports.showName=function()
    {
        console.log("name");
    }
`</pre>

**2.NPM:`包管理工具(类似:Maven,Composer)`**

简单使用:

下载个jquery来看看: npm install jquery

然后我们会发现在当前目录下,会出现一个node_modules文件夹.在次文件夹下会有jquery.这就是我们刚刚下载下来的jquery模块(jquery源码)

引用刚刚下载的jquery模块

<pre>`    var abc=require("jquery");//只要写文件夹名就行.它会自动寻找当前目录下的node_modules下面的文件夹
    console.log(abc);
`</pre>

**3.我们来自己创建个模块**

在node_modules文件夹下创建目录sxin(随便写)

然后我们把slib.js移动到sxin目录下

修改aa.js

<pre>`    var getlib=require('sxin');
    getlib.showName();
    console.log(getlib.age);
`</pre>

执行.我们会发现 报错 Error: Cannot find module 'sxin'

原来他加载模块会默认查找文件夹下的index.js,我们把aa.js改成index.js

然后在执行。好了 没问题

好  既然这边 那jquery 应该也是有个index.js吧？但是我们去看下发现并没有index.js。 这是什么鬼?

不是说要index.js的么?   我们会发现在jquery目录下有一个package.json .如果有这个文件，会根据这个文件去读取配置。

好，我们在sxin目录下也建一个package.json.

<pre>`{
    "main": "slib.js"
}

然后把刚才的index.js改回slib.js，然后在运行aa.js. 成功！
