PHP魔鬼训练班：第五回合：我得了懒癌

Posted on 2016-09-11  |  2016-11-14   |   In PHP , PHP魔鬼训练班

回顾一下上节课

我们专门新建了一个godconfig类,设置两个属性
1. prj_name(项目名)
2. prj_author(作者)

然后通过获取标准输入(stdin)把结果保存在类里面

问题来了

如果我是个懒货,新建了godconfig类后上了个厕所.忘记写那两个属性了

  1. 那么程序是否能运行？答案是依然可以.PHP会自动给你的类增加属性
  2. 这时我的”懒癌”又犯了,我甚至连godconfig类都不想建立了,可以吗？
    当然可以. $gc=new stdClass();

stdClass

这个PHP内置顶层类,实际上是一个基类,空白的.什么都没有

如果你临时用用的,其他地方很少用.那么根本不需要新建什么实体类,直接new 一个stdclass就可以了.

godinit

&lt;?php
    require('godconfig');

    class godinit
    {
        static $v="god version is 1.2";
        static function init()
        {
            $gc=new stdClass();
            echo "input your project name?".PHP_EOL;
            $gc-&gt;prj_name=trim(fgets(STDIN));
            echo "input your project author?".PHP_EOL;
            $gc-&gt;prj_author=trim(fgets(STDIN));

            return initConfig(json_encode($gc));
        }
        //xxoo
`</pre>

此时我们已经可以把godconfig删掉了

# 实战需求来了

我们在函数文件里写个方法,譬如叫

function TC() // 就是temp class的意思.代表返回一个临时类(这个类,因为懒 我不高兴新建成一个具体的文件,就是这么任性)

函数接受一个参数,参数是一个PHP数组,形式如下：

$f=array('prj_name'=>xxx,'prj_author'=>xxx);

这是一种带Key的数组,循环时:

foreach($f as $key=>$v) //   这里不懂请百度或强记

#### 第一种写法

god_func7

<pre>`function TC($arr)
    {
        $get_class=new stdClass();
        foreach($arr as $k=&gt;$v)
        {
            $get_class-&gt;$k=$v;
        }
        return $get_class;
    }
`</pre>

god

<pre>`static function init()
        {
            echo "input your project name?".PHP_EOL;
            $prj_name=trim(fgets(STDIN));
            echo "input your project author?".PHP_EOL;
            $prj_author=trim(fgets(STDIN));

            return initConfig(json_encode(TC(array('prj_name'=&gt;$prj_name,'prj_author'=&gt;$prj_author))));
        }
`</pre>

#### 第二种写法

god_func7

<pre>`function TC($arr)
    {
        return (object)$arr;
    }
`</pre>

#### 第三种写法

PHP5版本不支持

god_func7

<pre>`function TC($arr)
    {
        return new class($arr){
            function __construct($arr)
            {
                foreach($arr as $k=&gt;$v)
                {
                    $this-&gt;$k=$v;
                }
            }
        };
    }
`</pre>
