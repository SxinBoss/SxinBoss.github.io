## 正文

这节课我们快速接触下类,PHP也具备足够完善的面向对象机制.譬如如下代码:

<pre>`class GodInit
{
    public $god_version="";
    function init(){...}
}
`</pre>

类：人类自我发明的一种开发方式,是代码可读性更强,他不会提高程序性能性能。仅仅是为了"方便"

今天我们首先学习静态方法.静态属性的写法(暂时不要考虑静态和非静态在内存、堆栈之类的区别,PHP和JAVA有本质区别,别想太多)

譬如有个类叫 GodInit
1\. 拥有静态属性 static $god_version
2\. 拥有静态方法 static function init()

godinit

<pre>`&lt;?php
    class godinit
    {
        static $VERSION="god version is 1.1";
        static $prj_name="";//项目名称
        static $prj_author="sxin";//项目作者
        static function init()
        {
            echo "input your project name?".PHP_EOL;
            self::$prj_name=trim(fgets(STDIN));
            echo "input your project author?".PHP_EOL;
            self::$prj_author=trim(fgets(STDIN));

            return initConfig(self::$prj_name,self::$prj_author);
        }
    }
`</pre>

## 附加知识点:

以下内容如果不理解,完全不要紧!

大家有没有发现,这么多文件拷贝到/usr/local/bin 太low了

那么今天我们再学一个php打包的功能.如果不懂,请照抄.

<pre>`$pchar=new Phar("god.phar");
$pchar-&gt;buildFromDirectory(dirname(__FILE_));
$pchar-&gt;setStub($pchar-&gt;createDefaultStub('god'));
$pchar-&gt;compressFiles(Phar::GZ);
`</pre>

PS:如果保错
`Fatal error: Uncaught UnexpectedValueException: creating archive "god.phar" disabled by the php.ini setting phar.readonly`

则需要修改php.ini

<pre>`[Phar]
phar.readonly = Off
`</pre>

生成好之后我们来运行

`php god.phar -v`

`god version is 1.1`
没问题.ok
